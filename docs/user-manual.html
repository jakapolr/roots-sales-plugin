TOR Response Factory

A gate-enforced pipeline turning a raw TOR into a locked, submission-ready bid package

Why this exists Roots lost a bid by 0.14% — a fraction that traces to a missed scoring criterion or a weak evidence link. The factory ensures every mandatory requirement is covered and every scored criterion has a planned response before anyone writes a word of the proposal.

The one rule that matters most

G2 Matrix Freeze — no drafting before this gate passes The compliance matrix (TOR_Requirements) AND scoring matrix (Scoring_Matrix) must both be complete and frozen. The tor-factory-orchestrator enforces this and will block the draft step if G2 is not passed.

Single-entry point

For any active TOR, the orchestrator tells you exactly where you are and what to do next:

"orchestrate TOR [tor_id]"
"ทำ TOR ครบวงจร — [โครงการ]"
"factory mode — TOR-2026-003"

Never fabricate rule

Every TOR factory skill follows this: if a requirement is unreadable, evidence is missing, or a value is unknown — the skill writes Needs human review. It never invents requirements, guesses at evidence, or claims a capability Roots cannot prove.

The 10 Registers

Markdown files in Google Drive — the shared memory of the TOR factory

Location in Google Drive 8. Sales and Marketing/TOR/registers/
Skills read and write these through the Google Drive MCP. If Drive is not connected, skills output the updated rows and ask you to paste them.

TOR_Opportunities

Master pipeline — every TOR with buyer, budget, deadline, bid decision, and current gate. The primary index all other registers link to via tor_id.

Owner: roots-tor-intake

TOR_Requirements

Compliance matrix — one row per requirement, classified by type (mandatory/scored/legal…), with owner, evidence link, and response status.

Owner: roots-compliance-matrix

Scoring_Matrix

Weighted scoring plan — each evaluation criterion mapped to response angle, evidence, deck slide, and strength. The proposal and deck are both generated from this register.

Owner: roots-scoring-matrix

Company_Documents

Freshness control — every company/eligibility document with issue date, expiry, and status (valid/expiring/expired/missing). Never let a หนังสือรับรองบริษัท slip past 6 months.

Owner: roots-doc-freshness

CV_Master

Staff CV register — role, certifications, relevant projects, last updated, and approval status. A CV older than 90 days is flagged as stale.

Owner: roots-cv-builder

Evidence_Library

Reusable proof library — past project references, client letters, case studies, certificates. Grows after every closed project.

Owner: roots-evidence-matcher

LC_Bank_Facility

Bank guarantee capacity — facility limit, committed amount, remaining, expiry. Green/yellow/red eligibility for each bid's required bond.

Owner: roots-lc-check

Submission_Checklist

File-level submission state — generated, signed, stamped, reviewed, uploaded, printed for every document in every folder. The G5 lock depends on this being complete.

Owner: roots-submission-packager

Review_Log

QA issues — severity (🔴🟡🟢), owner, deadline, resolved status. G4 gate blocks while any 🔴 high is unresolved. 🔴 can only be waived by Executive.

Owner: tor-qa-reviewer

Lessons_Learned

Post-bid results — score, gap, root cause, reusable improvement, owner. Closes the loop so a 0.14% loss is never silently forgotten.

Owner: roots-lessons-learned

TOR Factory Skills

9 skills that run the G0–G6 pipeline

G0 roots-tor-intake

TOR Intake

Register a new TOR opportunity. Creates the tor_id, Drive folder tree (14 subfolders), and Google Calendar deadline gates for G1–G5. Run this within 2 hours of a TOR arriving.

Triggers

"เพิ่ม TOR ใหม่ — [ชื่อโครงการ] — [หน่วยงาน] — deadline [date]"
"intake TOR — [project name]"
"มี TOR มาใหม่ [paste basic info]"

Outputs

  • New row in TOR_Opportunities register (gate = G0)
  • Drive folder: TOR_[Buyer]_[Project]_[YYYYMMDD]/ with 14 subfolders
  • 6 Calendar events: G1 Bid/No-Bid through G5 Submission
G2 🔴 Critical roots-compliance-matrix

Compliance Matrix

Decompose the full TOR into a row-per-requirement compliance matrix. Every requirement gets a type, mandatory flag, owner, evidence pointer, and risk rating. No proposal drafting until this is complete.

Triggers

"ทำ compliance matrix — TOR-2026-003"
"แตก requirement ทั้งหมดจาก TOR นี้"
"requirement matrix — [paste TOR text]"

Requirement types classified

TypeWhat it meansRisk if missed
mandatory-passfailFails = disqualification🔴 Bid rejected
scoredWorth points in evaluation🔴/🟡 Score lost
legalEligibility / company documents🔴 Disqualification
commercialPrice, bond, payment terms🟡 Negotiation risk
deliveryTimeline, milestones, warranty🟡 Contractual
presentationSlides, demo, oral requirement🟡 Score loss
G2 🔴 Critical roots-scoring-matrix

Scoring Matrix (Criteria Optimizer)

Turn the TOR's evaluation criteria into a weighted response plan. Each criterion gets a response angle, evidence link, owner, deck slide number, and strength rating. The proposal and the presentation deck are both generated from this one register — they can never diverge.

Triggers

"ทำ scoring matrix — TOR-2026-003"
"วิเคราะห์เกณฑ์ให้คะแนน — [paste evaluation section]"
"จะได้คะแนนเต็มยังไง — criteria optimizer"

Strength ratings

RatingMeaningProjected capture
strongRoots has proven evidence for this criterion~90%+ of max score
mediumPartially covered — evidence thin or indirect~70%
weakGap — needs new evidence or honest downgrade~50%

G1 roots-lc-check

LC / Bank Guarantee Check

Computes remaining bank guarantee / LC facility against the bid's required bond amount. Returns green / yellow / red. A 🔴 stops the bid before wasting preparation time.

Triggers

"ตรวจ LC — TOR-2026-003 — bond 2% of 3.5M"
"LC เพียงพอไหม — bid bond [amount]"
"check bid bond capacity"

Eligibility logic

ResultConditionAction
🟢 GreenRemaining ≥ required + 10% buffer, facility not expiringProceed
🟡 YellowSufficient but thin margin, or facility expiring soonExecutive approval required
🔴 RedInsufficient, or facility expired/expiring before submissionStop — Director decision
Timeline note: Bank guarantee takes 3–5 business days. The skill computes the latest request date automatically.
G2 support roots-cv-builder

CV Builder

Checks CV_Master for freshness, drafts update requests for stale staff CVs, and formats role-matched tender CVs in Thai government procurement standard.

Triggers

"เตรียม CV ทีม — TOR-2026-003"
"CV สำหรับประมูล — [ชื่อโครงการ]"
"CV builder — roles: PM, SA, Developer"
A CV is stale if last_updated > 90 days. The "ทักษะที่เกี่ยวข้อง" section is tailored to each TOR's specific requirements — generic CVs score poorly.
G5 roots-submission-packager

Submission Packager

Generates the Submission_Checklist from TOR requirements and the 14-folder structure. Tracks generated / signed / stamped / reviewed / uploaded / printed for every file. The G5 lock only passes when all required rows are reviewed and zero 🔴 issues are open.

Triggers

"เตรียมซอง — TOR-2026-003"
"pack submission folder"
"G5 lock check — TOR-2026-003"
Physical copies: Every photocopy needs "รับรองสำเนาถูกต้อง" + ตรายาง — tracked separately from the original.
G6 roots-lessons-learned

Lessons Learned

Records the bid result, score gap, root cause, and reusable improvement in Lessons_Learned. Closes the TOR loop and feeds insights forward into future bids.

Triggers

"บันทึก lessons learned — TOR-2026-003 — lost — score 82.36 vs 82.50"
"G6 — bid result — [project] — won"
"ผลประมูลออกแล้ว — [paste result]"
Even wins get a G6 entry — document what worked so it's repeated intentionally, not by accident.

Odoo Custom Skills

Roots-specific skills for analysis, estimation, and bid qualification

G1 roots-tor-analyzer

TOR Analyzer

Reads a TOR PDF (OCR via PDF Tools MCP for scanned documents) and extracts all structured information — basic info, technical requirements, qualification requirements, risk flags — then scores Roots fit 1–10 and returns a Go/No-Go recommendation. On GO, outputs the next factory steps.

Triggers

"analyze this TOR [paste text or upload PDF]"
"should we bid on this? [paste TOR content]"
"วิเคราะห์ TOR นี้ [หน่วยงาน + โครงการ]"

Fit scoring

DimensionWhat it measures
Technical fitCan Roots deliver the technical scope?
Qualification fitDoes Roots meet all mandatory qualifications?
Competitive positionIs this winnable?
Commercial fitIs the budget viable for the scope?

Average ≥ 7 → GO. Below 7 → No-Go or discuss with Director.

Compares Odoo 18 Enterprise vs Community (BEECY) for a specific client — module coverage, cost, Thai localization needs. Produces a structured recommendation table.

Triggers

Estimation roots-manday-estimator

Manday Estimator

Estimates mandays and project cost (THB) for any Odoo scope. Returns three scenarios: Conservative / Target / Stretch with cost at 40% margin. Always adds 25% buffer for government bids.

Triggers

"ประเมิน cost สำหรับ [client] — [scope]"
"how much should we quote for [project]?"

Blended rate: THB 5,000/day  |  Standard margin: 40%

Agents

Run in a separate context — longer, stateful tasks that must not flood the main conversation

Skills vs Agents Skills fire inline in the conversation. Agents spawn their own context, have access to specific tools (Read / Write / etc.), and return a summary when done. Use agents when the task involves multiple documents, long reasoning, or writing to Drive files.
Orchestrator tor-factory-orchestrator

TOR Factory Orchestrator

The pipeline manager for the full G0–G6 cycle. Reads the registers to assess which gate the bid is actually at, verifies pass criteria are met, and tells you exactly what to do next. Enforces the G2 matrix-freeze rule — will refuse to allow drafting until G2 is passed.

Tools: Read, Write

"orchestrate TOR TOR-2026-003"
"factory mode — where are we on [project]?"
"ทำ TOR ครบวงจร — [โครงการ]"

Hard rules it enforces

  • Never allow drafting before G2 PASS
  • Never allow G5 lock with open 🔴 issues
  • LC must be checked before G1 GO
  • All mandatory docs must be valid before G2
  • G6 must happen after every bid result

TOR QA Reviewer

Compliance coverage QA agent. Reads TOR_Requirements, Scoring_Matrix, and the proposal draft. Checks 6 dimensions and returns severity-classified issues. The G4 gate blocks while any 🔴 is open.

Tools: Read only

"ตรวจ compliance proposal TOR-2026-003 [paste/link proposal]"
"QA proposal — TOR-2026-003"
"coverage check — did we answer every criterion?"
